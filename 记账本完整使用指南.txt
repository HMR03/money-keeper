━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
第一步：电脑上预览确认
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1. 解压下载的 zip 文件
  2. 直接双击 index.html，浏览器会打开记账本
  3. 试试点 ＋ 记几笔账，确认功能正常
  4. 确认没问题后，继续下面的部署步骤

  注意：本地预览时 Service Worker 不生效（需要 HTTPS），
  这不影响功能，部署后就正常了。


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
第二步：注册 GitHub 账号
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  如果你已经有 GitHub 账号，跳到第三步。

  1. 打开 https://github.com/signup
  2. 输入邮箱 → 设置密码 → 输入用户名
  3. 验证邮箱（GitHub 会发一封确认邮件，点邮件里的链接）
  4. 注册完成，记住你的用户名，后面要用


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
第三步：创建仓库并上传文件
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1. 登录 GitHub 后，点右上角的 ＋ 号 → 选 "New repository"

  2. 填写信息：
     - Repository name：输入 money-keeper（或你喜欢的英文名字）
     - 选择 Public（必须是 Public 才能用 GitHub Pages）
     - 其他都不用勾
     - 点绿色按钮 "Create repository"

  3. 在新建的空仓库页面，你会看到一段提示
     往下找到 "uploading an existing file" 这个蓝色链接，点它

  4. 上传文件：
     - 打开你电脑上解压出来的文件夹
     - 全选以下文件和文件夹，拖进 GitHub 的上传区域：

       index.html
       manifest.json
       sw.js
       icons（整个文件夹，里面有两个 png 图标）

     - 不要上传「部署指南.txt」，那个是给你看的

  5. 页面下方 "Commit changes" 区域
     - 直接点绿色按钮 "Commit changes"
     - 等待上传完成

  6. 上传完成后，检查仓库结构应该是这样的：

     money-keeper/
     ├── index.html
     ├── manifest.json
     ├── sw.js
     └── icons/
         ├── icon-192.png
         └── icon-512.png

     ⚠️ 如果 icons 文件夹里的文件不对（比如多套了一层文件夹），
     需要删掉重新上传，确保路径正确。


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
第四步：开启 GitHub Pages
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1. 在你的仓库页面，点顶部的 "Settings"（齿轮图标）

  2. 左侧菜单找到 "Pages"，点进去

  3. "Build and deployment" 区域：
     - Source：选 "Deploy from a branch"
     - Branch：选 "main"，后面的目录选 "/ (root)"
     - 点 "Save"

  4. 等 1-2 分钟，刷新页面
     顶部会出现一个绿色横幅，写着：
     "Your site is live at https://你的用户名.github.io/money-keeper/"

  5. 点击这个链接，确认记账本能在浏览器里正常打开

  6. 记住这个网址，这就是你的记账 App 永久地址：
     https://你的用户名.github.io/money-keeper/

  ⚠️ 如果页面显示 404：
     - 等 3-5 分钟再试（GitHub Pages 部署需要时间）
     - 检查 index.html 是否在仓库根目录（不是某个子文件夹里）


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
第五步：自己手机安装
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  【iPhone 安装】

  ⚠️ 必须用 Safari 浏览器！微信内置浏览器和 Chrome 不支持。

  1. 打开 Safari
  2. 在地址栏输入你的 GitHub Pages 网址
  3. 等页面完全加载好
  4. 点 Safari 底部正中间的 分享按钮
     （一个方框中间有个向上箭头的图标 ⬆️ ）
  5. 在弹出的菜单里，往下滑动，找到 "添加到主屏幕"
     （图标是一个 ＋ 号，在列表比较靠下的位置）
  6. 在弹出的页面里：
     - 名称默认是「记账本」，可以改成你想要的名字
     - 点右上角 "添加"
  7. 回到 iPhone 主屏幕，就能看到一个橙色的「¥」图标了
  8. 点开它，全屏运行，没有 Safari 的地址栏和工具栏
     看起来和用起来跟原生 App 一样

  卸载方式：长按图标 → 移除 App


  【安卓手机安装】

  1. 打开 Chrome 浏览器
  2. 在地址栏输入你的 GitHub Pages 网址
  3. 等页面加载完成后，Chrome 可能会自动弹出
     "将记账本添加到主屏幕" 的提示，直接点安装
  4. 如果没有自动提示：
     - 点右上角三个点 ⋮
     - 选 "添加到主屏幕" 或 "安装应用"
     - 确认
  5. 主屏幕出现图标，点开即用

  卸载方式：长按图标 → 卸载 / 移除


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
第六步：分享给别人使用
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  你的记账 App 网址是：
  https://你的用户名.github.io/money-keeper/

  任何人拿到这个链接，都能安装到自己手机上。
  每个人的数据独立存在各自手机本地，互不干扰。

  【分享方式一：直接发链接】

  把网址通过微信、QQ、短信、邮件发给对方。
  对方收到后：
  - iPhone：用 Safari 打开 → 分享 → 添加到主屏幕
  - 安卓：用 Chrome 打开 → 添加到主屏幕

  ⚠️ 微信里直接点开链接会用微信内置浏览器打开，
  这个浏览器不支持"添加到主屏幕"。
  告诉对方点右上角 ··· → "在 Safari 中打开"（iPhone）
  或 "在浏览器中打开"（安卓），然后再操作。


  【分享方式二：生成二维码（推荐）】

  让别人扫一下就能打开，不用手动输网址。

  方法 1：微信生成
  - 用电脑浏览器打开你的记账 App 网址
  - 用微信扫一扫右上角 → 选「提取二维码」
  - 或者直接把网址发到微信对话里，长按链接可以生成二维码

  方法 2：在线工具
  - 打开 https://cli.im（草料二维码）
  - 粘贴你的网址
  - 点生成，下载二维码图片
  - 可以把这个图片发给任何人

  方法 3：Chrome 浏览器
  - 用 Chrome 打开你的记账 App 网址
  - 点击地址栏右边的分享图标（或右键地址栏）
  - 选「创建二维码」
  - 保存或截图

日常使用指南
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  【记账】
  - 打开 App → 点中间的 ＋ 按钮
  - 选 支出/收入 → 选分类 → 输入金额 → 写备注(可选) → 保存
  - 保存后自动回到流水页面

  【查看流水】
  - 首页就是流水页，按日期分组显示
  - 顶部卡片显示当月总支出、总收入、结余
  - 左右箭头可以切换月份
  - 点击任意一条记录可以编辑或修改

  【查看报表】
  - 底部点「报表」tab
  - 上面是收支趋势折线图（最近 12 个月）
  - 下面是支出分类排行（哪个分类花最多）
  - 可以切换「月」和「年」视图

  【自定义分类】
  - 底部点「设置」→ 支出分类 / 收入分类
  - 可以添加新分类、删除不需要的分类

  【数据备份（建议每月做一次）】
  - 设置 → 导出数据 → 保存 JSON 文件
  - 文件可以存到手机的"文件"App、iCloud、或发给自己的微信

  【恢复数据】
  - 设置 → 导入数据 → 选择之前导出的 JSON 文件
  - 会覆盖当前数据，适合换手机或数据丢失时使用

更新 App
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  如果以后需要修改代码（修 bug 或加功能）：

  方法一：直接编辑
  1. 打开你的 GitHub 仓库页面
  2. 点击要修改的文件（比如 index.html）
  3. 点右上角铅笔图标（编辑）
  4. 修改内容
  5. 点 "Commit changes" 保存

  方法二：重新上传
  1. 仓库页面点 "Add file" → "Upload files"
  2. 把修改过的文件拖进去，覆盖旧的
  3. 点 "Commit changes"

  更新后：
  - 手机端下次打开会自动更新（需要联网）
  - 如果没有自动更新，可以：
    iPhone：删掉主屏幕图标，重新用 Safari 添加
    安卓：在 App 设置里清除缓存
  - 如果还不行，打开 sw.js 文件，把第一行的
    'moneykeeper-v1' 改成 'moneykeeper-v2'，重新上传

  ⚠️ 更新 App 代码不会丢失记账数据！
  数据存在手机本地的 IndexedDB 里，跟代码文件是分开的。

数据安全说明
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  - 所有记账数据存在手机本地，不会上传到任何服务器
  - GitHub Pages 只托管代码文件（HTML/CSS/JS），不接触数据
  - 换手机前记得导出数据，在新手机上导入恢复
  - 清除浏览器数据会导致记账数据丢失，建议定期导出备份

  推荐的备份方式：
  1. 导出 JSON 文件
  2. 保存到以下任意位置（越多越安全）：
     - iPhone 的"文件"App
     - iCloud Drive / 百度网盘
     - 微信"文件传输助手"（发给自己）
     - 邮件附件发给自己
     - 电脑上存一份

常见问题
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Q: 手机上打开是空白页？
  A: 检查 GitHub Pages 是否已生效（Settings → Pages 看状态），
     等 3-5 分钟再试。确认网址末尾有 / 斜杠。

  Q: Safari 分享菜单里没有 "添加到主屏幕"？
  A: 往下滑，这个选项在列表比较靠下的位置。
     如果真的没有，确认你用的是 Safari 而不是其他浏览器。
     微信内置浏览器不行，点右上角 ··· → 在 Safari 中打开。

  Q: 添加到主屏幕后打开还是有地址栏？
  A: 确认 manifest.json 已正确上传，
     且里面的 "display": "standalone" 没有被改动。
     有时需要删掉图标重新添加。

  Q: 换了新手机数据怎么办？
  A: 旧手机：设置 → 导出数据 → 保存 JSON 文件
     新手机：安装 App → 设置 → 导入数据 → 选择 JSON 文件

  Q: GitHub Pages 有费用吗？
  A: 完全免费，没有任何费用。仓库设为 Public 即可。

  Q: 我的记账数据别人能看到吗？
  A: 不能。代码是公开的（别人能看到 App 的代码），
     但数据存在每个人自己的手机上，GitHub 上没有任何用户数据。

  Q: 不用了能删除吗？
  A: 手机上长按图标删除 App。
     如果要彻底删除，去 GitHub 仓库 Settings → 最下面
     → Delete this repository，输入仓库名确认删除。

  Q: 断网能用吗？
  A: 第一次打开需要联网加载。之后 Service Worker 会缓存页面，
     断网也能正常记账。下次联网时会自动同步代码更新。
